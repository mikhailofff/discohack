# AltDrive: Yandex Disk FUSE Client

**AltDrive** — это производительный и функциональный клиент для Linux, позволяющий монтировать облачное хранилище Яндекс Диск как локальную файловую систему через FUSE. Приложение реализует концепцию «файлов по запросу»: структура облака видна сразу, но данные скачиваются только при обращении к ним.

---

## 📂 Архитектура и структура проекта

Проект разделен на модули для обеспечения гибкости и упрощения отладки:

| Файл | Роль в системе |
| :--- | :--- |
| **`service.py`** | **Центральный узел.** Управляет GUI (PyQt6), хранением конфигурации и жизненным циклом FUSE-процесса. |
| **`engine.py`** | **Движок FUSE.** Основная логика файловой системы. Маппит системные вызовы (`getattr`, `read`, `write`) на API-запросы. |
| **`adapter.py`** | **Сетевой уровень.** Инкапсулирует работу с Yandex Disk REST API. Реализует стриминг данных и обработку ошибок сети. |
| **`cache.py`** | **Менеджер кэша.** RAM-кэш для метаданных (атрибуты файлов, списки папок) для мгновенного отклика проводника. |
| **`test_fuse.py`** | **CLI-инструмент.** Позволяет запускать монтирование из терминала напрямую, передавая токен и лимиты аргументами. |
| **`auth.py`** | **Модуль авторизации.** Содержит TCP-сервер для автоматического перехвата OAuth-кода из браузера. |

---

## 🚀 Основные возможности

* **Умное кэширование (LRU):** Локальный кэш данных ограничен пользователем. При достижении лимита (например, 5 ГБ), система автоматически удаляет старые файлы, к которым давно не обращались.
* **Отложенная синхронизация (Write-back):** Когда вы сохраняете файл на смонтированном диске, он сначала записывается в локальный кэш, а в облако уходит только после закрытия дескриптора файла (событие `release`).
* **Автоматический OAuth 2.0:** Не нужно вручную копировать токены. Приложение открывает браузер, и после подтверждения прав само получает токен через локальный порт 8080.
* **GUI на PyQt6:** Удобное управление через системный трей: статус подключения, сброс сессии и настройка параметров кэша.
* **Многопоточность:** FUSE-движок работает в отдельном потоке (`QThreadPool`), что предотвращает «зависание» интерфейса при тяжелых сетевых операциях.

## 🛠 Установка и зависимости

### 1. Системные пакеты
Для работы FUSE в Linux необходимо установить соответствующие библиотеки:
```bash
# Для Ubuntu / Debian / Mint
sudo apt install fuse3 libfuse-dev
```

### 2. Зависимости Python
Установите необходимые пакеты через `pip`:
```bash
pip install PyQt6 requests fusepy urllib3
```
## 💻 Использование

### Способ 1: Запуск через GUI (Рекомендуется)

1. Запустите основной сервис:
```bash 
python service.py
```
2. В системном трее появится иконка диска.

3. Откройте «Параметры кеша» и укажите:

* mountpoint: папка, куда будет примонтирован диск (например, `~/YandexCloud`).
* cache: папка для временных файлов.
* limit: максимальный объем кэша в МБ или ГБ.
4. Нажмите «Войти в Диск». Приложение откроет страницу Яндекса в браузере.
5. После подтверждения диск будет автоматически примонтирован.

### Способ 2: Запуск через терминал (CLI)
Если вам нужно запустить драйвер без графической оболочки:
```bash 
python test_fuse.py /path/to/mount --token <ВАШ_ТОКЕН> --limit 10 --cache /tmp/alt_cache
```

## ⚙️ Технические детали для разработчиков

**Жизненный цикл файла** 
* **getattr / readdir**: Engine проверяет RAM-кэш (`cache.py`). Если данных нет, запрашивает их через `adapter.py`.
* **read**: Если файла нет в локальном кэше на диске, `adapter.py` скачивает его чанками.
* **write**: Данные пишутся в локальную копию в папке кэша. Файл помечается как `dirty`.
* **release**: Если файл был изменен (`dirty`), он загружается обратно на Яндекс Диск методом PUT.

### Конфигурация
Все настройки и токен доступа сохраняются в JSON-файл в домашней директории: `~/.cloud_bridge_config.json`

## ⚠️ Ограничения и рекомендации

* **Пустая директория**: Точка монтирования должна быть пустой перед запуском.
* **Безопасность** : Не передавайте файл `.cloud_bridge_config.json` третьим лицам, так как он содержит полный доступ к вашему Диску.
* **Принудительное размонтирование**: Если приложение закрылось некорректно, выполните:
```bash
fusermount -u /путь/к/точке/монтирования
```
**AltDrive** — свободная альтернатива официальному клиенту для тех, кто ценит контроль и экономию дискового пространства.
